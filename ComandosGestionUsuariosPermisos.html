Comandos De Gestión De Usuarios Y Permisos

Los comandos de LINUX permiten al usuario controlar el sistema desde la interfaz de línea de comandos (CLI) en lugar de utilizar una interfaz con iconos. Estos son instrucciones de texto introducidas en la terminar para indicarle al sistema operativo exactamente que es lo que se tiene que hacer. Los comandos que se ingresan en la terminal de LINUX se distinguen entre mayúsculas y minúsculas, además de seguir una sintaxis.

Los comandos de gestión de usuarios y permisos permiten controlar quién tiene acceso a archivos y qué operaciones pueden realizar. Esta categoría es clave para la seguridad del sistema, ya que ayuda a definir y ajustar los permisos de lectura, escritura y ejecución, y a gestionar los usuarios y grupos.

sudo

Este comando ejecuta comandos con privilegios de superusuario.

Sintaxis

sudo [comando]

Opciones comunes:

  • -l: lista los comandos permitidos o prohibidos para el usuario actual.
  • -v: actualiza el tiempo de autenticación, extendiendo el tiempo sin pedir la contraseña.
  • -k: invalida la autenticación, forzando a pedir la contraseña la próxima vez.
  • -u [usuario]: ejecuta el comando como otro usuario, no como root.

Ejemplo

sudo apt-get update — Ejecuta apt-get update con privilegios de superusuario.

useradd

Este comando agrega un nuevo usuario.

Sintaxis

useradd [opciones] [usuario]

Ejemplo

sudo useradd juan — Crea un usuario llamado juan.

passwd

Este comando cambia la contraseña de un usuario.

Sintaxis

passwd [usuario]

Ejemplo

passwd pedro — Cambia la contraseña del usuario pedro.

chown

Este comando cambia el propietario de un archivo o directorio

Sintaxis

chown [propietario:grupo] [archivo]

Ejemplo

sudo chown juan:juan archivo.txt — Cambia el propietario y grupo de archivo.txt a juan.

groups

Muestra los grupos a los que pertenece un usuario

Sintaxis

groups [usuario]

Ejemplo

groups — Muestra los grupos a los que pertenece el usuario actual.
