Gestión de Usuarios y Permisos

El concepto de usuario en Linux permite separar entornos de ejecución para diferentes propósitos. Dos personas pueden trabajar simultáneamente en el mismo sistema, teniendo cada uno un usuario diferente, y un directorio personal diferente.

También es muy común que muchos servicios internos del sistema tengan su propio usuario para restringir el acceso de ese servicio como mecanismo de seguridad. De este modo, si un servicio ve su seguridad comprometida por un ataque, el acceso que tenga el usuario de ese servicio servirá como contención del ataque, y no podrá acceder a ficheros pertenecientes a otro usuario (de persona o servicio).

Comandos útiles de Linux para la gestión de usuarios

Los comandos de administración de usuarios de Linux son sencillos y permiten a los administradores realizar operaciones básicas y avanzadas, como crear, modificar y eliminar cuentas y grupos de usuarios, y administrar propiedades de cuentas. Algunos comandos de Linux útiles que los administradores usan con regularidad son:

Dominio Descripción
usuarioadd

Cree nuevas cuentas de usuario de Linux, especificando varias opciones como nombre de usuario, directorio de inicio e ID de usuario.
password

Establecer o cambiar la contraseña de una cuenta de usuario.
usuariodel

Eliminar una cuenta de usuario, garantizando que el usuario ya no tenga acceso a los recursos del sistema.
grupoadd

Agregue un nuevo grupo en Linux y especifique las opciones del grupo.
grupodel

Eliminar un grupo existente y todos los archivos asociados a ese grupo.
chown

Transferir la propiedad de un archivo o directorio a un usuario o grupo diferente.

Creación y gestión de cuentas de usuario

Para crear una nueva cuenta de usuario en Linux:

1. Abra la línea de comando y escriba el comando useradd [opciones] NOMBRE DE USUARIOseguido del nombre de usuario. Este comando crea una nueva cuenta de usuario con configuraciones predeterminadas, como un directorio de inicio y un ID de usuario. También puede especificar diferentes opciones para personalizar estas configuraciones.

2. Después de crear la cuenta, puede establecer o cambiar la contraseña escribiendo el comando passwd [opciones] NOMBRE DE USUARIO.

3. Modifique varios atributos de una cuenta de usuario utilizando el comando usermod [opciones] NOMBRE DE USUARIO, incluido el cambio del nombre de usuario, el directorio de inicio, el ID de usuario o el ID de grupo.

4. Para eliminar una cuenta de usuario, escriba el comando userdel [opciones] NOMBRE DE USUARIO seguido del nombre de usuario. Esto elimina la cuenta de usuario del sistema y garantiza que el usuario ya no tenga acceso a los recursos del sistema.

5. Utilice el comando “chown” para cambiar la propiedad de un archivo o directorio y el comando archivo chmod [opciones] para modificar los permisos.

Configuración de permisos de usuario y control de acceso

La configuración adecuada de los permisos de usuario y el control de acceso en Linux garantiza que los usuarios tengan acceso adecuado a los recursos del sistema manteniendo al mismo tiempo la seguridad e integridad del sistema.

Permisos de los usuarios en Linux

En Linux, cada archivo y directorio está asociado con tres tipos de permisos: lectura, escritura y ejecución (designados por r, w y e).

La lectura permite al usuario leer el contenido del archivo, pero no cambiarlo. La escritura permite que el usuario modifique el archivo. La ejecución, como el nombre lo dice, permite que el usuario ejecute el archivo, en caso de ser ejecutable.

Pero sucede que los permisos no funcionan aisladamente, o sea, de forma que el usuario tenga permiso de lectura o de escritura o de ejecución. Los permisos funcionan en conjunto. Esto quiere decir que cada archivo/directorio tiene los tres permisos establecidos, corresponde al dueño determinar cuál de estos permisos está habilitado para los usuarios o no.

Puede ser que una determinada cantidad de usuarios tengan permiso para modificar un archivo, pero otros no, por ejemplo. De ahí la necesidad de usar grupos. En este caso, el permiso de escritura de este archivo será dada al grupo, por lo que todo usuario miembro puede cambiar el archivo. Ten en cuenta que es necesario tener cierto cuidado con los permisos. Por ejemplo, el que informa de que el usuario tenga permiso de escritura si no tiene permiso de lectura habilitada.

Ahora que ya sabemos el significado de las divisiones de la cadena, vamos a entender lo que las letras r, w, x, y el carácter – representan:

  • r: significa permiso de lectura (read).
  • w: significa permiso de escritura (write).
  • x: significa permiso de ejecución (execution).
  • –: significa permiso deshabilitado.

Estos permisos se pueden conceder o restringir para el propietario, el grupo y otros. De forma predeterminada, el propietario tiene permisos de lectura, escritura y ejecución, mientras que el grupo y otros solo tienen permisos de lectura y ejecución.

Para modificar los permisos, puede utilizar el comando chmod seguido de las opciones deseadas. Por ejemplo, para otorgar permisos de lectura y escritura al propietario, puede utilizar el comando:
chmod u+rw archivo.txt

Para eliminar los permisos de lectura y escritura del grupo, utilice el comando: chmod g-rw archivo.txt

Linux también admite listas de control de acceso (ACL), que brindan un control más detallado sobre el acceso de los usuarios. Las ACL permiten especificar permisos de acceso para usuarios individuales o grupos, anulando los permisos predeterminados.

Para configurar las ACL, puede utilizar el comando “setfacl” seguido de las opciones deseadas. Por ejemplo, para otorgar permisos de lectura y escritura a un usuario específico, puede utilizar el comando:
setfacl -mu:nombre de usuario:rw archivo.txt

Configurar cuidadosamente los permisos y el control de acceso garantiza que los usuarios tengan el nivel adecuado de acceso a los recursos del sistema y al mismo tiempo mantiene la seguridad de su sistema Linux.

El propietario tiene permisos de lectura, escritura y ejecución, el grupo y otros pueden leer y ejecutar el archivo.

El archivo puede ser leído, escrito y ejecutado por quien sea

Solo el propietario puede leer el archivo, pero ni el mismo puede modificarlo o ejecutarlo y ni el grupo ni otros pueden hacer nada en el.

El usuario propietario puede leer y escribir, el grupo puede leer el archivo y otros no pueden hacer nada.
